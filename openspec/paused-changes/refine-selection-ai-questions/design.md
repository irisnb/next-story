## Context

当前生成链路已经由 Rust 后端在 `src-tauri/src/llm_config/generate.rs` 集中组装固定 system message，随后把冻结选区原文作为唯一 user message 发给已保存的唯一 LLM。现有提示只要求“提出问题和几个思考方向”，尚未规定模型如何阅读极短创作种子、具体叙事片段或介于两者之间的材料。

第一阶段仍然只看见局部选区，无法了解作者想完成的完整故事，也无法可靠判断人物长期轨迹、世界观、主线或主题。本设计只改善局部提问质量，不把缺失的全局认知伪装成已经解决。

## Goals / Non-Goals

**Goals:**

- 通过一个后端固定方法提示词，引导模型从当前选区提出有材料依据、清楚而不武断的问题。
- 让模型面对抽象材料时偏向打开可能性，面对具体材料时偏向观察已有张力，材料模糊时自然混合两种阅读动作。
- 让问题数量和展开深度服从材料承载的信息，不机械凑数。
- 让回复大体呈现“简短总览—分点展开—简短收束”的可读层次，同时允许模型按材料自然调整。
- 保持策略集中、可测试、可回退，并允许第二阶段整体替换或扩展。

**Non-Goals:**

- 不摄入、摘录或检索罗伯特·麦基著作，不建设完整或缩小版知识库。
- 不加入选区前后文、全文摘要、世界观、人物、剧情主线、价值或作者意图等作品级认知。
- 不加入多轮对话、用户追问、会话历史、流式输出或停止生成。
- 不建立 `seed`、`scene` 等持久材料类型，不新增分类 API、用户模式选择或模型调用。
- 不让 AI 代写、评分、判定创意优劣，或直接改动草稿本和正文本。

## Decisions

### 1. 在现有 system message 中承载第一阶段方法内核

后端继续发送两条消息：固定 system message 和原样 selected text 的 user message。固定方法提示词负责说明助手职责、局部阅读姿态、可选观察角度、表达边界和回复层次。

选择这一方式，是因为现有链路已经具备集中组装、密钥隔离和固定请求测试；第一阶段无需增加第二次分类调用、检索层或前端参数。备选方案是先分类再生成，但它会引入额外延迟、错误路径和固定类别，且当前没有失败证据证明其必要。

### 2. 阅读姿态是生成指导，不是正式分类

提示词描述连续倾向：抽象材料更多向外打开，具体材料更多贴文观察，兼具两者的材料自然混合。模型不需要输出材料类型，系统也不保存分类结果。

不按字符数、关键词、人名、引号或格式建立硬规则。“背叛”可以是创作种子，而单字对白“滚”也可能是具体行动；机械阈值会错误约束材料。

### 3. 麦基方法只压缩为局部观察入口

提示词可以引导模型留意人物表达与行动、意图与阻力、表达与回应、局面或关系变化，以及抽象概念内部的价值张力。这些入口不是必答清单，也不在用户回复中展示理论名称。

选择“观察入口”而非理论摘要，是为了让方法服务具体材料，而不是让 AI 输出写作课程或用单一戏剧观审判作品。

### 4. 明确观察与开放问题分工

模型可以提出有力度的暂定观察，再把观察转化为用户可以判断的问题；不得把推断写成确定事实，也不得用“如何加强”“哪里不足”等问法预设作品有缺陷。边界说明集中在回复收束处，不要求每个分点反复使用“可能、也许、仅供参考”等削弱表达的词。

### 5. 结构是可读倾向而非僵硬模板

回复应先概括本次打开的方向，再分点说明观察与问题，最后简短收束。问题数量不固定；极短材料可以只产生一个有力度的问题，信息较多的材料可以产生多个彼此有区别的问题。

不新增结构化 JSON 响应或前端分区渲染。AI 面板继续显示纯文本，避免为了首版排版扩大协议和 UI 范围。

### 6. 用请求契约测试和代表性人工样例验证

自动测试继续锁定：请求只有固定 system message 与选区原文、无额外上下文、非流式单次调用，并检查方法提示词包含关键职责与边界。由于真实模型输出具有不确定性，不能用精确字符串测试宣称“问题质量已经保证”；应使用抽象词、极短具体表达和较完整片段做人工观察记录，识别套话、机械凑数、武断推断和代写倾向。

## Risks / Trade-offs

- [不同兼容模型对提示词遵循程度不同] → 锁定请求契约，并以代表性真实模型样例观察行为；不把提示词当作绝对控制器。
- [固定提示词过长会稀释选区并增加成本] → 只保留局部观察动作、表达边界和输出层次，不加入理论历史、术语解释或书中案例。
- [方法提示可能把所有材料戏剧化] → 明确观察入口是可选的，允许材料只是意象、停顿、气氛或尚未形成叙事的种子。
- [“总—分—收”变成机械格式] → 只要求可读层次，不固定标题、问题数或逐项配额。
- [第一阶段局部问题与完整故事意图不一致] → 明确系统只依据当前选区，不声称理解选区之外的故事；作品级认知留待第二阶段。
- [当前策略限制未来升级] → 不新增持久分类、公开 API 或前端模式；第二阶段可替换 system message 组装方式而无需迁移用户数据。
