## Context

当前仓库处于新项目重启早期，只有项目宪法、方向文档、反面案例和 OpenSpec 工作流文件，还没有应用代码。第一版必须先证明最小写作闭环：桌面应用能新建作品、打开作品、编辑草稿本和正文本、保存后重新打开仍能回读。

本项目的最高产品边界是：草稿本和正文本只由用户亲手产生内容；AI 不能直接插入、替换、改写、删除或移动这些文本。本 change 不引入 AI 能力，只建立后续能力所依赖的桌面地基和用户文本事实源。

后续 AI 面板要想好用，需要一份内置 AI 内容库种子文档，用原创浓缩的剧作提问方法论保证 AI 提问和发散质量。该文档非常重要，但它不属于本桌面地基 change；它应在后续 AI 面板 / AI 质量相关 change 中与用户高度互动确认。

## Goals / Non-Goals

**Goals:**

- 使用 Tauri 建立桌面应用地基，先在 Windows 验证，代码结构不主动锁死 macOS / Linux。
- 使用 TypeScript 构建前端界面，Rust 作为中等核心，负责本地作品文件结构、读取、保存和校验。
- 提供简单欢迎页：新建作品、打开作品。
- 采用“一个作品 = 一个文件夹”的本地存储模型。
- 将用户可见文本和系统数据分成两个抽屉：`作品文本/` 与 `next-story-system/`。
- 提供草稿本和正文本两个标签页，打开作品后默认进入草稿本。
- 手动保存时同时保存草稿本和正文本。

**Non-Goals:**

- 不实现 AI 面板、LLM 调用、模型配置、AI 历史记录。
- 不创建内置 AI 内容库种子文档；该文档属于后续 AI 相关 change，并且创建时必须与用户高度互动确认内容。
- 不实现自动备份、版本快照、恢复 UI。
- 不实现富文本、专业剧本排版、Fountain 解析、自动格式化。
- 不实现最近作品列表、项目模板、删除作品、导入任意文本。
- 不实现正式版系统目录隐藏或迁移；开发期使用可见的 `next-story-system/`。

## Decisions

### 1. Desktop shell: Tauri + TypeScript + Rust core

使用 Tauri 作为桌面壳，TypeScript 负责界面，Rust 核心负责本地文件能力和结构校验。

理由：用户明确愿意接受学习成本，优先追求稳定、简洁、边界清晰和模块化。Tauri 的前端/Rust 命令分层能把文件写入能力集中在 Rust 核心，而不是让界面任意读写本地文件。

替代方案：Electron + TypeScript 更快、更成熟，但更依赖团队纪律来防止 UI、文件、状态和未来 AI 能力混在一起。

### 2. Rust core owns project file operations

前端不直接读写作品文件。前端通过 Tauri 命令请求 Rust 核心执行：创建作品、打开作品、校验结构、读取草稿本和正文本、保存两个本子。

理由：写入用户文本事实源是核心边界，必须集中管理。后续加入 AI 时，Rust 核心可以继续不提供“AI 写入草稿本/正文本”的命令。

### 3. Project folder structure

开发期作品结构为：

```text
<作品文件夹>/
├─ 作品文本/
│  ├─ 草稿本.txt
│  └─ 正文本.txt
└─ next-story-system/
   └─ project.json
```

`作品文本/` 是用户可见抽屉，存放用户直接理解的文本文件。`next-story-system/` 是开发期可见系统抽屉，存放项目元信息。自动备份、AI 会话、模型配置等未来目录不在本 change 创建。

理由：中文文件名更符合用户理解；Rust 使用路径 API 拼接路径，可以安全处理中文路径。系统抽屉开发期可见，方便学习和调试。

### 4. Minimal project metadata

`project.json` 只记录第一版地基需要的最小元信息，例如：作品名、创建时间、更新时间、结构版本。它不保存草稿本或正文本内容。

理由：文本事实源应保持在 `作品文本/草稿本.txt` 和 `作品文本/正文本.txt`，避免元信息文件变成第二文本事实源。

### 5. Manual save saves both notebooks

保存按钮或 Ctrl+S 同时保存草稿本和正文本。

理由：第一版规则越简单越好。避免“当前页保存了，另一个页没保存”的分裂状态。

## Risks / Trade-offs

- [Risk] Tauri/Rust 比 Electron 学习成本高。→ Mitigation: Rust 核心保持克制，只做本地项目结构、读写和校验；不把 UI 状态或复杂业务塞进 Rust。
- [Risk] 中文路径在实现中被错误地字符串拼接。→ Mitigation: Rust 层统一使用路径 API，测试中包含中文作品名和中文文件名。
- [Risk] 作品文件夹被用户手动改坏。→ Mitigation: 打开作品时只做结构校验；结构不合法则提示，不自动修复。
- [Risk] 第一个 change 范围继续膨胀。→ Mitigation: 明确排除 AI、自动备份、设置页、版本快照和复杂编辑功能。
- [Risk] 后续 AI demo 因缺少高质量提问方法论而变成普通聊天。→ Mitigation: 将内置 AI 内容库种子文档标记为后续 AI 面板 / AI 质量 change 的关键前置，但不纳入本 change 实现任务。
