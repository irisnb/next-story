## Context

项目目前有四份承担不同职责的文档：`方向/核心方向宪章.md` 是长期北极星，`方向/第一版方向共识-2026-07-01.md` 是第一版取舍依据，`AGENTS.md` 是协作宪法，`README.md` 是公开入口和当前状态说明。现有内容已经反复确认 AI 不代写、判断权属于用户，但“为什么存在、降低什么门槛、知识怎样服务创作者”的表达尚未形成清晰的权威链。

本次已经确认的使命是：把认真讲述自己故事的权利交给每一个想创作的人。这里的“权利”是产品使命，不是法律权利或结果保证；工具降低开始创作和获得帮助的门槛，但不替代专业训练、生活经验、审美、思考、实践与坚持。

## Goals / Non-Goals

**Goals:**

- 建立四份文档从长期使命到公开状态的单向权威关系。
- 完整表达知识、AI 和创作者之间的关系，并把创作者的内容权、解释权、评价权和方向权写清楚。
- 统一“帮助用户重新看见故事”等使命级用语，避免把故事降格为等待系统优化的材料。
- 保持 README 的实现事实准确，明确区分长期使命、当前已实现和未来规划。
- 让以后新增 AI 感官、知识和行动能力时，都能依据稳定使命检查是否越权。

**Non-Goals:**

- 不改变应用行为或已经归档的功能事实。
- 不实现第一阶段方法库、方法卡、知识检索、作品上下文或多轮对话。
- 不承诺用户无需学习、无需思考即可写出好故事，也不承诺把专业创作者能力转移给用户。
- 不把任何创作理论写成所有故事必须遵循的客观规律。
- 不在四份文档中逐字复制完整使命全文。

## Decisions

### 1. 使用分层权威链，而不是四份平级使命文档

`核心方向宪章` 完整解释使命和永久权力关系；`第一版方向共识` 将其转成第一版范围、非目标和验证标准；`AGENTS.md` 提炼为参与者必须遵守的红线；`README.md` 结合长期使命与正式规格，形成面向读者的短摘要和状态说明。

选择这一结构，是为了让使命只在一个上游位置完整定义，同时让各类读者在不跳转文档时仍能看到必要边界。备选方案是四处复制确认稿，但它会造成修改时的措辞漂移。

### 2. 将使命、能力边界和结果承诺分开

使命可以说“把认真讲述自己故事的权利交给每一个想创作的人”；能力边界只承诺降低开始创作和获得帮助的前置门槛；结果边界明确不保证专业能力、统一创作水平或公认优秀作品。

这样既保留产品理想，也避免把创作民主化偷换成自动化生产。

### 3. 将知识定义为内部服务资源，而非用户资格考试

四份文档按各自粒度表达同一原则：“知识不是用来证明 AI 懂得多，而是用来减少用户必须先懂多少。”知识最终应帮助 AI 给出贴合故事的观察、问题和可能性，而不是展示术语、授课或裁判作品。

本 change 只确定知识的产品位置，不决定未来知识系统的存储、检索或组合架构。

### 4. 将创作者权力写成永久治理边界

核心宪章完整定义内容权、解释权、评价权和方向权；第一版方向说明这些权力如何体现在 AI 面板和用户亲手写入中；AGENTS 将其转成可执行红线；README 只保留公众必须知道的短摘要。

AI 可以提供观察、假设和不同方向，但不得垄断解释、判定创作优劣、决定故事方向或直接写入草稿本和正文本。

### 5. 灵魂与肉身只作为方向模型，不成为当前技术架构

核心宪章可以用简图说明：灵魂决定 AI 为谁服务和知识如何使用；肉身由感官、记忆、思考和行动能力构成；AI 打开可能，创作者作出判断并亲手完成。第一版方向只引用这一模型解释当前极小能力，AGENTS 和 README 不展开完整哲学论证。

该模型不得被误写成模块命名、数据结构或未来架构承诺。

### 6. 按上游到下游顺序更新并横向复核

实施顺序固定为 `核心方向宪章 → 第一版方向共识 → AGENTS.md → README.md`。最后检查使命与状态是否分离、术语是否唯一、四项权力是否一致、永久文本边界是否保留，以及 README 是否仍只把正式规格能力写成已实现。

## Risks / Trade-offs

- [使命表述过大，被读成效果保证] → 在使命后紧邻能力边界和不承诺事项，明确降低门槛不等于转移专业能力。
- [四份文档重复后再次漂移] → 完整论证只放核心宪章，下游只保留职责所需摘要和链接。
- [方向文档污染实现事实] → README 继续以 `openspec/specs/` 为已实现真相源，使命与规划使用明确状态标签。
- [“知识服务用户”被误读为理论必须隐形] → 只规定默认以普通语言服务创作，不禁止用户主动了解来源或理论；具体交互留待未来 change。
- [“灵魂与肉身”被误当技术设计] → 明确它是产品方向模型，不派生当前模块或接口。
- [新增红线与既有铁律重名] → 优先整合现有“判断权”“AI 不写文本”等条目，不创建第二套同义概念。

## Migration Plan

1. 先更新长期方向真相源，再向下同步第一版、协作宪法和公开说明。
2. 不迁移数据、不修改应用；若复核发现使命与当前事实混淆，可按 Git diff 整体回退本纯文档 change。
3. 归档时新增治理规格，并将 README 增量合并到现有 `project-readme` 正式规格。

## Open Questions

无。本次只固化已经由用户确认的使命与能力边界；第一阶段方法库和未来知识系统继续留在暂停提案及后续讨论中。
