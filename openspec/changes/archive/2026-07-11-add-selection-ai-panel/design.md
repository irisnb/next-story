## Context

当前编辑器使用草稿本和正文本两个 `textarea`，已经具备保存、未保存保护和从编辑器进入 LLM 配置后返回的状态保留。Rust 侧已有唯一 LLM 配置、结构化 URL 校验和真实 `chat/completions` 连接测试，但没有面向创作的生成命令。前端也没有选区快照、浮动召唤入口或 AI 面板。

本 change 是第一条 AI 纵向闭环。首版故意只发送用户选中的原文，不试图用任意字数的前后文冒充上下文方案。设计重点是让最小功能真实可用，同时把确定会变化的责任边界隔离开，避免将来替换编辑器、增加上下文或升级会话形式时推倒重写；但不提前创建插件系统、多 Agent、Prompt 注册中心或空的上下文框架。

## Goals / Non-Goals

**Goals:**

- 从鼠标或键盘形成的有效选区产生与 `textarea` 解耦的不可变 `SelectionSnapshot`。
- 在选区附近提供唯一“召唤 AI”入口，并遵守类似 Word 选区上下文工具的出现、定位和消失规则。
- 由 Rust 生成用例集中组装固定首版思考任务，前端只提交选区原文。
- 复用已保存的唯一 LLM 配置完成一次真实非流式生成，并提供可读成功或错误结果。
- 在右侧可收起 AI 面板中显示临时材料，结构上不向 AI 链路提供任何草稿本或正文本写入能力。
- 将选区适配、请求协调、Rust 生成用例、生成客户端和面板状态分开，使后续扩展可以替换单一责任模块。

**Non-Goals:**

- 不读取或发送选区前后文、全文、摘要、项目元数据、AI 内容库或其他隐式上下文。
- 不做多轮对话、会话历史、重启恢复、流式输出、停止生成、请求队列或并行会话。
- 不做 Markdown/HTML 渲染、自动复制、插入、替换、一键应用或任何 AI 写入用户文本。
- 不替换当前 `textarea` 编辑器，不实现通用编辑器插件体系。
- 不增加多 provider、多模型槽位、模型预设列表或前端传入 API Key 的新调用路径。

## Decisions

### 1. 用稳定快照隔离编辑器实现

编辑器适配层把当前活选区转换为：

```text
SelectionSnapshot
├── notebook: draft | manuscript
├── selectedText: string
├── start: number
└── end: number
```

快照在用户点击“召唤 AI”时冻结。`start/end` 用于验证快照身份和后续适配，不发送给模型。`manuscript` 只作为明确对应“正文本”的代码层标识，不作为第二个产品名称。当前适配器可以读取 `textarea.selectionStart`、`selectionEnd` 和 `value`，但请求和面板模块只能依赖 `SelectionSnapshot`，不能读取 DOM 编辑控件。

选择这一边界，是因为编辑器内核以后可能替换；只要新编辑器能够产生同一快照，AI 主链路无需改变。首版不创建通用插件注册表，只有一个 `textarea` 适配器。

### 2. 有效选区与浮动入口使用独立控制器

选区经 `trim()` 后至少包含一个字符才有效，但快照保留用户选中的原始文本，不主动裁剪、改写或静默截断。鼠标选择和键盘选择统一通过 selection 变化产生候选快照。

浮动按钮以 `selectionEnd` 对应的焦点端作为首版唯一锚点。锚点位于 `textarea` 内容视口内时显示，滚出内容视口时隐藏；滚动后若仍可见则重新定位。选区为空、只含空白、点击编辑器其他位置或切换本子时隐藏。点击按钮前已经保存候选快照，点击后冻结该快照并隐藏活入口；面板展开或收起不会让旧入口重新出现，必须由新的有效选择重新触发。

按钮控制器只负责“当前是否存在可提交的活选区”，不发网络请求。

### 3. 固定 Prompt 只由 Rust 生成用例组装

前端只把 `SelectionSnapshot.selectedText` 交给窄用途 Tauri command。Rust 生成用例把固定职责和用户原文组装为模型消息：围绕选区提出帮助创作者继续思考的问题，并给出几个可能的思考方向，不代写正文。

固定指令只存在于 Rust 生成用例，不散落在 DOM 事件、前端桥接或底层 HTTP 模块中。后续上下文、摘要或思考方式若获得独立规格，可以扩展该用例的输入与消息组装；首版不放置空的 `nearbyText`、`summary` 或知识库字段。

### 4. 生成命令只接收选区文本，后端自行加载唯一配置

前端调用窄用途 `generate_ai_thinking` command，只传本次选中的原文。Rust command 从应用本地数据目录加载已保存的唯一 LLM 配置；配置不存在或不完整时返回可辨识、可读错误，不允许前端把 API Key 随请求重复传入。

生成 HTTP 层复用现有安全 URL 解析、HTTPS/loopback 规则、禁止重定向和超时基础，但连接测试与创作生成是两个明确用例。生成请求使用 `stream: false`，提取第一个合法非空 assistant 文本并返回纯字符串。

新 command 使用窄的可序列化错误契约 `GenerateAiError { code, message }`。`code` 至少区分 `configuration_required`、`authentication`、`timeout`、`network`、`request_too_large`、`service` 和 `invalid_response`；`message` 是经过安全清洗的中文说明。前端只依据 `code` 切换状态，不解析文案。错误不得回传 Authorization、API Key、请求正文或任意完整远端响应。

### 5. 请求协调首版采用可替换的单请求策略

前端 AI 请求协调层持有唯一进行中 Promise。生成期间编辑器保持可写、可切换本子和可形成新选区，但浮动召唤入口禁用，不能发起第二次请求。成功或失败后释放请求锁。

首版不取消旧请求、不排队。`single-in-flight` 通过根本不执行第二个 client 调用来避免结果乱序；不建立不可能发生的并行结果排序系统。该策略不属于 Rust 生成用例、模型客户端或面板业务的永久规则。

### 6. AI 面板用显式状态和 replace-current 策略

面板状态拆成两个正交维度：

```text
visibility: open | closed
request:
  idle
  | loading(snapshot)
  | success(snapshot, response)
  | error(snapshot, error)
  | configuration-required(snapshot)
```

收起只修改 `visibility`，不改变 `request`；请求完成只更新 `request`，不得自动展开已经收起的面板。首版 `replace-current` 策略在新请求开始时用新快照替换当前请求内容，并在成功后替换当前回复；该策略集中在面板状态模块，不散落操作 DOM。以后可由独立 change 升级为对话或历史，而不修改选区或生成命令。

面板默认约 420px。正常桌面宽度与编辑器并排并压缩编辑区；窄窗口使用覆盖式抽屉。收起保留会话内内容，重新展开恢复；应用重启不恢复。

### 7. 纯文本渲染和零写回采用结构性限制

AI 回复使用 `textContent` 或等价纯文本绑定，保留换行并允许选择复制，不通过 `innerHTML` 解析模型内容。面板只接收结果对象，不持有 `saveProject`、编辑器 DOM 写入函数或任何“应用到正文”回调。该边界通过模块依赖和行为测试共同验证，不以脆弱的源码字符串搜索代替测试。

这比只靠按钮文案保证边界更可靠：即使未来修改视觉层，首版 AI 模块也没有直接改动两个本子的能力。

### 8. 配置缺失和普通错误分开呈现

缺少保存配置时，后端返回稳定的配置缺失类别；面板进入 `configuration-required`，保留快照并显示“前往配置”。返回编辑器后不自动请求，由用户点击“重新请求”使用原快照明确发起。

其他失败按稳定错误 `code` 进入 `error`，显示安全中文信息和“重新请求”。不自动重试，不因用户产生新选区改变重试输入。配置页返回继续复用现有“不破坏写作现场”机制。

### 9. 请求结果受当前作品生命周期隔离

请求协调器记录发起请求时的作品实例令牌。返回欢迎页或成功替换作品时使该令牌失效；迟到的成功或错误只能被忽略，不能重新打开面板、覆盖新作品的面板状态或影响任何本子。进入 LLM 配置页不卸载作品，因此可以保留当前请求状态，但不得发起第二个调用。

## Risks / Trade-offs

- **[Risk]** `textarea` 没有直接提供像富文本编辑器那样稳定的选区 DOM 矩形。 → **Mitigation:** 首版只定位 `selectionEnd` 焦点端；纯函数测试覆盖快照和可见性判定，镜像字体、行高、padding、border、换行、滚动和 DPI 的真实几何由 Tauri 人工验收，不用 Node 假布局冒充证明。
- **[Risk]** 用户选中很长文本导致模型拒绝请求。 → **Mitigation:** 首版不静默截断；原样发送并将请求过长错误可读地返回面板，后续根据真实使用设计上下文预算。
- **[Risk]** 用户生成期间继续编辑，旧回答可能与当前文本不再一致。 → **Mitigation:** 面板明确显示冻结的“本次选区”；请求永远基于点击时快照，不跟随当前编辑器变化。
- **[Risk]** 模型输出包含 HTML 或脚本片段。 → **Mitigation:** 只按纯文本渲染，不解析 HTML/Markdown。
- **[Risk]** 模块化演变成提前设计通用平台。 → **Mitigation:** 只实现一个适配器、一个 Rust 生成用例、一个生成 client、一种请求策略和一种显示策略；未来能力必须单独 propose。
- **[Trade-off]** 只给选区可能让模型回答缺少语境。 → **Mitigation:** 本 change 只验证可靠地传递选区并获得临时思考材料；上下文策略是后续独立产品问题，不用任意字数暂时代替。

## Migration Plan

1. 新增模块和 Tauri command，不改变作品文件、现有 LLM 配置文件或草稿本/正文本格式。
2. 若需要回退，可移除 AI 入口、面板和生成 command；现有写作、保存与 LLM 配置能力保持不变。
3. 当前不持久化 AI 状态，因此没有 AI 会话数据迁移或清理任务。
